# data/ —— 各智能体的原始数据

每个智能体一个子目录：`data/<agent>/`。工具与 prompt 统一通过
`tradeflow.data_loader.load_data(agent, name)` 读取，**禁止在代码里硬编码这些内容**。

- 表格（`.csv` / `.xlsx`）→ `load_data` 返回 `list[dict]`（每行一个字典，键为表头）
- 文档（`.md`）→ `load_data` 返回原始字符串
- 文件缺失 → 返回空（`[]` / `""`），不崩，方便"代码先行、数据后补"
- 声明了 `required_columns` 但表里缺列 → **显式报错**（不静默出错）

CSV 请存为 **UTF-8（建议带 BOM）**，Excel 双击能正常显示中文，`load_data` 以
`utf-8-sig` 读取会自动去掉 BOM。

## 数据 schema（G2：列名 / 类型 / 示例，收数据前先对齐）

### compliance / 禁词表_<站点>.csv
按站点分文件（`禁词表_US.csv` / `禁词表_UK.csv` …）。

| 列 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|
| 违规词 | str | ✓ | `best` | 命中即拦截，大小写不敏感；可为多词短语 `100% cure` |
| 类型 | str | ✓ | `极限词` | 极限词 / 医疗夸大 / 虚假认证 / 侵权 … |
| 合规替代 | str |  | `high-quality` | 命中时给出的合规改写建议 |
| 说明 | str |  | `US 站禁用` | 备注、风险等级等 |

### compliance / 品牌IP黑名单.csv
| 列 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|
| 名称 | str | ✓ | `AirPods` | 他人品牌/商标/专利话术，命中即侵权风险 |
| 类型 | str |  | `商标` | 商标 / 品牌 / 专利话术 |
| 合规替代 | str |  | `wireless earbuds` | 命中时的合规改写 |
| 说明 | str |  | `苹果注册商标` | 备注 |

### compliance / 类目风险.csv
| 列 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|
| 类目 | str | ✓ | `医疗器械` | 与输入类目做包含匹配 |
| 风险等级 | str | ✓ | `高` | 高 / 中 / 低 |
| 风险类型 | str |  | `审核+侵权` | 风险来源 |
| 说明 | str |  | `需资质，涉 FDA` | 备注 |

### compliance / 白名单.csv
| 列 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|
| 短语 | str | ✓ | `bestnest` | 自有品牌词/已确认合规表达；`compliance_gate` 中落在该短语内部的违规词放行 |
| 说明 | str |  | `自有注册品牌名` | 备注 |

### listing / 关键词词根库.csv
| 列 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|
| 词根 | str | ✓ | `shockproof` | 待埋入标题/五点的关键词 |
| 类目 | str | ✓ | `手机壳` | 与输入类目做包含匹配 |
| 搜索量 | int |  | `12000` | 用于排序，越大越优先 |
| 优先级 | str |  | `高` | 高/中/低，辅助取词 |

### imagery / 图片规范.csv
| 列 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|
| 规则 | str | ✓ | `无文字水印` | 规范条目名 |
| 适用 | str | ✓ | `主图` | 主图 / 场景图 / 通用 |
| 违规关键词 | str |  | `文字;水印;促销` | 分号分隔；在图片描述里命中即判违规 |
| 说明 | str |  | `主图不得叠加文字` | 规则详情，也进 checklist |

| 列 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|

| ASIN | str | ✓ | `B0HOT0CASE1` | 待拆解的竞品 ASIN |
| 备注 | str |  | `类目头部精品` | 说明 |

### market / 关键词市场数据.csv
| 列 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|

| 列 | 类型 | 必填 | 示例 | 说明 |

### selection / 评分权重.csv（**占位，待老板校准 G3**）
`维度,权重,说明` —— 维度取 市场容量/竞争宽松度/毛利/侵权安全/改良空间，权重之和≈1。

### selection / 选品门槛.csv（**占位**）
`项,值,说明` —— 含 `最低毛利分`、`最低侵权安全分`、`禁做品类`（分号分隔）。命中即淘汰。

### ads / 搜索词报告_*.xlsx|csv（#4 广告优化）
亚马逊后台「商品推广 → 搜索词报告」原样导出（中文列名），粒度为 **按天 × 搜索词**。
`*_真实.*` 含店铺真实数据，已 gitignore，只在本地使用；仓库放 `搜索词报告_样例.csv`。

| 列 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|
| 广告活动名称 | str | ✓ | `火把手动广泛 6.7` | 用于映射到 SKU |
| 客户搜索词 | str | ✓ | `solar torch lights` | 聚合与分类的主键 |
| 展示量/点击量/花费 | num | ✓ | `420 / 12 / 9.6` | 基础指标 |
| 7天总销售额 / 7天总订单数(#) | num | ✓ | `95.97 / 2` | 转化指标 |
| 广告投入产出比 (ACOS) 总计 | float |  | `0.15` | **无销售时为空**，解析需兜底 |

### ads / 结算报告_*.xlsx|csv
亚马逊「付款 → 日期范围报告」交易明细（英文列名；真实 xlsx 每月一个 sheet）。
只取 `type=Order` 行，按 SKU 汇总回款率 = total / product sales。

| 列 | 类型 | 必填 | 示例 |
|---|---|---|---|
| type | str | ✓ | `Order`（其余 Refund/Adjustment 忽略） |
| sku / quantity / product sales / total | mixed | ✓ | `SAMPLE-TORCH-4P / 1 / 31.99 / 19.99` |

### ads / 广告活动映射.csv
`广告活动名称,SKU,备注` —— 广告活动对应哪个 SKU（搜索词报告里没有 SKU 列，须人工维护）。
未映射的活动用 `调词规则.csv` 的「默认盈亏ACOS」兜底。

### ads / 成本表.csv（**占位，替换为真实成本**）
`SKU,采购成本,头程运费,说明`（单位 USD/件）。
盈亏平衡 ACOS = 回款率 −（采购成本+头程)/客单价，缺成本时退化为回款率的保守折扣。

### ads / 调词规则.csv（**阈值外置，G3 待老板校准**）
`参数,值,说明` —— 三分类阈值（零转化点击数/ACOS 倍数/最低订单数）与调价幅度。

### monitor / 监控清单.csv（竞品监控）
`ASIN,平台,站点,备注` —— 要持续盯的竞品；平台/站点留空默认 amazon/US。

### monitor / 监控规则.csv（**阈值外置**）
`参数,值,说明` —— 价格变动预警%、评论新增预警条数。
快照落 `data/monitor/快照/*.json`（本地产物，gitignore）。
